Guild config

This page will help you set up the mmr commands for your server, as well as explain some of the other guild settings. Note that you must have the MANAGE GUILD permission to modify guild settings.

Here's a list of all the guild settings and a brief description of what they do

GameBoards1 - The ID of a lounge on gb.hlorenzi.com.
GameBoards2 - The ID of a lounge on gb.hlorenzi.com. This is for when you have a seperate leaderboard for rt's and ct's.
Spreadsheet1 - The google spreadsheet ID of your server's leaderboard.
Spreadsheet2 - The google spreadsheet ID of your server's leaderboard. This is for when you have a seperate leaderboard for rt's and ct's.
SheetName - The sheet name of the sheet with all player stats on your leaderboard. Defaults to Leaderboard
RatingName - The name of the points system your server uses. Defaults to MMR
PlayerIndex - The index of the column on your spreadsheet containing player names.
RatingIndex - The index of the column on your spreadsheet containing elo/mmr/point values.
StatIndexes - The indexes of the columns on your spreadsheet you want to include for the stats command..

Hlorenzi's Game Boards

If your leaderboard is on gb.hlorenzi.com, then setting up the commands for it is very simple. All you need is the ID of your lounge.
First, get the link to your lounge. We'll use this one as an example:

https://gb.hlorenzi.com/reg/gWsHAI/

You'll want the letters after /reg/. So for this lounge, the ID would be:

gWsHAI

Once you have the ID, use the following command to set it in the guild settings (using your server's prefix): /changesetting GameBoards1 gWsHAI.

After you've set this up, you're all set! Both the mmr and stats commands should be working for your server.

If you have a 2 spreadsheets for rt's and ct's, follow the procedure above for the rt sheet, and do the same for the ct sheet, but use GameBoards2 instead of GameBoards1 in the command. You access the different leaderboards by doing mmr rt/ct <player>. Currently it is not possible to change the prefix for the two leaderboards from rt and ct.

It is important to note that the GameBoards guild setting takes priority over the Spreadsheet setting. So if you ever decide to switch to a Google Sheet leaderboard, reset both GameBoards settings with set reset <setting>.

Google Sheets leaderboard

If your leaderboard is on a google sheet, it's a little more complicated to set up, so follow the steps below.

First, you'll have to get the ID of the google spreadsheet that your leaderboard is on. To get it, first get the link to you spreadsheet, we'll use this one as an example:

https://docs.google.com/spreadsheets/d/1IPGK_kCgdqSLwcFjzgeLsRW7qV3MLCgcSBABdZHtK4o/edit?usp=sharing

You'll want the part between the /d/ and /edit. So for this spreadsheet, the ID would be:

1IPGK_kCgdqSLwcFjzgeLsRW7qV3MLCgcSBABdZHtK4o

Once you have the ID, use the following command to set it in the guild settings (using your server's prefix): /changesetting Spreadsheet1 1IPGK_kCgdqSLwcFjzgeLsRW7qV3MLCgcSBABdZHtK4o.

If you have a 2 spreadsheets for rt's and ct's, follow the procedure above for the rt sheet, and do the same for the ct sheet, but use Spreadsheet2 instead of Spreadsheet1 in the command. You access the different leaderboards by doing mmr rt/ct <player>. Currently it is not possible to change the prefix for the two leaderboards from rt and ct.

Setting the sheet name

Next, you'll want to setup the sheet name. This defaults to Leaderboard, so you may not have to change it. Let's take a look at a snippet from the leaderboard from the section above.

The part underlined is what should be set as the sheet name. If it's not Leaderboard, run this command to change it to what it is: /changesetting SheetName Leaderboard.

Setting the player and rating index

Afterwards, you'll want to setup the PlayerIndex and RatingIndex settings. Let's look at a snippet of the leaderboard from the first section again.

In this leaderboard, the player names are on the 3rd column. When setting the index in the guild settings, you'll want to take the column that the player names/point values are on and subtract by 1. So in this example, you would do: /changesetting PlayerIndex 2.

This also applies for the column with the point values. Here, they're on the 4th column, so you would do: /changesetting RatingIndex 3.
Once these values are set up properly, the mmr command should now be fully working. If you would also like the stats command working, read the next section.

Setting the stat indexes

Finally, if you want the stats command working, you'll want to set the StatIndexes setting. These should be indexes seperated by commas. We'll take a look at the leaderboard from the previous sections again.

From this leaderboard, I would want columns B to K, except C since that's the player column (even if you include the player column in this setting, the bot will ignore it).
Column B is the 2nd column, so it's index would be 1. Column F is the 6th column, so it's index would be 5. Same goes for the rest of the columns. In this example, the command I would run would be: /changesetting StatIndexes 1,3,4,5,6,7,8,9,10.
Once this setting is properly set, the stats command should be working.
